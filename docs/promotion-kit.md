## 传播时最该强调的 3 个点

1. 它解决的是 **上下文留存**，不是再造一个大平台
2. 它可以 **一条命令安装**
3. 它有 **prompt-native daily memory automation**，不是靠隐藏脚本假装有记忆

## 推荐配图顺序

当前先不要用旧占位 SVG。等真实截图补齐后，建议顺序是：

1. README 第一屏：一句话定位 + 一条命令安装。
2. 安装终端：`install.sh` 完成并提示 workspace 路径。
3. 工作区目录：`0-System/status/context/memory/about-me`。
4. Codex automation：每日记忆整理任务。
5. 飞书桥接或 skills：作为可选扩展展示。

## 评论区高频回复模板

### “这和只写一个 AGENTS.md 有什么区别？”

> `AGENTS.md` 更像规则入口，`codex-x` 额外给的是一整套可持续更新的记忆结构：`status / context / daily memory / long-term memory`，再加初始化器和可选桥接入口。

### “我不用飞书，值不值得看？”

> 值得。飞书 CLI 是可选模块，最核心的是 `workspace-template + create-codex-x + daily memory automation` 这条本地工作区线。

### “为什么不直接做成平台？”

> 现在刻意不走平台路线，是因为很多人真正缺的是一个能立刻用起来的本地工作区，而不是一套更重的系统。

### “现在最推荐先体验什么？”

> 先把安装提示词丢给 Codex，再看生成出来的工作区目录和每日记忆整理 automation。这样最快能感受到项目的核心价值。
