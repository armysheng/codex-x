# Final Launch Plan

这份文档不是素材库，而是**实际执行顺序**。

## 推荐顺序

### Day 0

1. 检查 GitHub 仓库首页是否正常展示：
   - README
   - 5 张展示图
   - description
   - topics
   - release

2. 准备发帖配图：
   - 首页总览图
   - 一键安装图
   - `doctor + smoke` 图

3. 先发中文主贴
   - 推荐用 `docs/channel-posts.md` 里“推荐先发这一版”

### Day 1

1. 发 X / Twitter 英文版
2. 发 V2EX 版
3. 在 GitHub Discussion 再发一遍简版

### Day 2-3

1. 收评论和 issue
2. 观察哪些地方最让人不理解
3. 优先修：
   - 安装入口
   - README 首屏理解成本
   - 截图是否足够说明问题

## 每条帖子最小配置

- 1 张总览图
- 1 张安装图
- 1 个最短安装命令
- 1 句“它解决什么问题”
- 1 个 GitHub 链接

## 先不要做的事

- 不要一开始发太长技术复盘
- 不要一开始讲太多架构词
- 不要把焦点放在“实现很优雅”

先让人明白：

1. 它解决什么问题
2. 为什么上手成本低
3. 为什么和别的方案不同

## 发布后观察什么

- GitHub star / traffic
- README 首页停留与点击
- 用户最常问的问题
- 是模板吸引人，还是飞书桥接吸引人

## 当前最推荐首发主轴

就围绕这 4 个词：

- memory-first
- one-command install
- codex-native
- optional feishu bridge
